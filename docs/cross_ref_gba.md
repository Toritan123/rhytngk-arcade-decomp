## Cross-Reference Methodology Limitations

- Arcade DTPKs use **higher quality** samples (44 kHz, often 16-bit PCM)
  while GBA uses **8-bit / 16 kHz**. Direct byte-level matching is not feasible.
- Arcade samples per DTPK average 132; GBA instruments per bank average 28.
  Arcade includes voice/announcer samples and effect variants that GBA omits.
- Arcade DTPK names are stage-based (`stage4_2`) not game-based, requiring
  manual cross-referencing with `seqsel` table to map DTPK → game.
- Future work: audio fingerprinting (FFT) to confirm jingle correlations,
  and parsing the SH-4 `seqsel` table to get DTPK → game name mapping.
